프로젝트 개요

엑셀 없이 Python 코드만으로 E2E 테스트를 작성하는 구조입니다. 역할이 세 레이어로 명확히 분리됩니다.

어디에 있나
pages/
→
무엇을 하나
pages/ 메서드
→
어떤 순서로
tests/
레이어파일담당
설정config/settings.pyURL, 타임아웃, 브라우저 옵션
공통기반pages/base_page.pyclick, type_text 등 Selenium 공통 메서드
페이지객체pages/xxx_page.py각 페이지 엘리먼트 locator + 동작 메서드
시나리오tests/test_xxx.py테스트 순서 + assert 검증
전역설정tests/conftest.py브라우저 생성/종료, 전역 예외처리
📁

폴더 구조

e2e_pom/
  ├── config/
  │   └── settings.py  ← URL, 타임아웃, 브라우저 옵션 설정
  ├── drivers/

  │   ├── login_page.py  ← 로그인 엘리먼트 + 동작
  │   └── search_page.py  ← 검색 엘리먼트 + 동작
  ├── tests/
  │   ├── conftest.py  ← 브라우저 fixture, 전역 예외처리
  │   ├── test_main.py  ← 메인 페이지 시나리오
  │   ├── test_login.py  ← 로그인 시나리오
  │   └── test_search.py  ← 검색 시나리오
  ├── pytest.ini  ← pytest 실행 설정
  └── reports/  ← 스크린샷, 리포트 저장

실행 방법

# 전체 테스트 실행
pytest

# 특정 파일만 실행
pytest tests/test_login.py

# 키워드 필터 (함수명에 "정상" 포함된 것만)
pytest -k "정상"

# 브라우저 지정
pytest --browser=firefox

# URL 지정
pytest --url=https://mysite.com

# HTML 리포트 생성 (pytest-html 설치 필요)
pytest --html=reports/report.html

# 실패해도 전체 계속 실행
pytest --tb=short

config/settings.py

브라우저 옵션, 타임아웃, 경로 등 전역 설정값을 관리합니다. 테스트 URL은 여기서 변경합니다.

BASE_URL = "https://example.com"  # 기본 테스트 URL

TIMEOUTS = {
    "explicit":  10,   # 엘리먼트 대기 최대 10초
    "page_load": 30,   # 페이지 로딩 최대 30초
}

PATHS = {
    "screenshots": "reports/screenshots",
}

엘리먼트 탐색

메서드설명반환

새 페이지 추가하기

예시: 상품 목록 페이지를 추가하는 경우

1단계 — pages/product_page.py 생성

from pages.base_page import BasePage

class ProductPage(BasePage):

    # ── 엘리먼트 (실제 사이트 HTML에 맞게 수정) ──────────────────
    PRODUCT_ITEM  = ("class", "product-card")
    CART_BTN      = ("id", "add-to-cart")
    SORT_DROPDOWN = ("id", "sort-select")
    CART_COUNT    = ("class", "cart-badge")

    # ── 동작 ──────────────────────────────────────────────────────
    def add_first_product_to_cart(self):
        self.click_nth(self.CART_BTN, 1)

    def sort_by(self, option: str):
        self.select_by_text(self.SORT_DROPDOWN, option)

    def get_product_count(self) -> int:
        return len(self.find_all(self.PRODUCT_ITEM))

    def get_cart_count(self) -> int:
        return int(self.get_text(self.CART_COUNT))

2단계 — tests/test_product.py 생성

import pytest
from pages.product_page import ProductPage

PRODUCT_URL = "https://mysite.com/products"

@pytest.fixture
def product_page(browser):
    page = ProductPage(browser)
    page.navigate(PRODUCT_URL)
    return page

class TestProduct:

    def test_상품목록_노출(self, product_page):
        assert product_page.get_product_count() > 0

    def test_장바구니_담기(self, product_page):
        before = product_page.get_cart_count()
        product_page.add_first_product_to_cart()
        after = product_page.get_cart_count()
        assert after == before + 1, "장바구니 수량이 1 증가해야 함"

def login(self, username, password):
    try:
        self.type_text(self.USERNAME, username)
        self.type_text(self.PASSWORD, password)
        self.click(self.LOGIN_BTN)
    except Exception as e:
        self.screenshot("login_fail")  # 실패 시 스크린샷
        raise RuntimeError("로그인 실패: {}".format(e))

tests 예외처리 패턴

def test_로그인(self, login_page):
    try:
        login_page.login("admin", "admin123")
        assert "dashboard" in login_page.get_current_url()

    except AssertionError:
        login_page.screenshot("assert_fail")
        raise

    except Exception:
        login_page.screenshot("unexpected_fail")
        raise
⚠ 주의 예외를 처리한 후 반드시 raise로 다시 던져야 pytest가 FAIL로 기록합니다.
raise 없이 except만 쓰면 테스트가 PASS로 처리되어 버그를 놓칩니다.
